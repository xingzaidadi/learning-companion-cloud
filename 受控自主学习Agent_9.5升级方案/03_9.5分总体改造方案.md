# 9.5 分总体改造方案

## 1. 总体目标

当前系统已经具备学习陪跑闭环，但仍是：

```text
规则编排 + 局部 AI 调用 + 本地兜底
```

9.5 分目标是升级为：

```text
受控自主学习 Agent
= LearningManagerAgent
+ 专业工具 Agent
+ 学生画像 Memory
+ 课本资料 RAG
+ 质量评估 Evaluator
+ 多轮 Tutor
+ 动态 Planner
+ 完整测试体系
+ 三端清晰 UI
```

## 2. 9.5 分定义

不是功能很多就 9.5，而是要满足以下条件：

| 维度 | 9.5 标准 |
|---|---|
| 功能可用 | 家长一次设目标，孩子每天可独立学，系统自动检查和补漏 |
| 易用 | 孩子端任意时刻只有一个主动作，家长端第一屏给结论，管理端三步发布 |
| Agent 智能 | 能观察、诊断、规划、执行、批改、记忆、调整 |
| 安全 | 不超纲、不直接给答案、不越权、不无限加任务 |
| 可解释 | 每个计划、题目、结论都有来源、证据、置信度 |
| 稳定 | AI 失败可降级，服务重启可恢复，重复操作不产生脏数据 |
| 测试 | 单元、接口、集成、E2E、长期、多日、AI 异常、权限、质量评估全覆盖 |

## 3. 改造主线

```text
第一条线：Agent 内核标准化
第二条线：学生画像和长期记忆
第三条线：动态排程和任务容量控制
第四条线：RAG 课本资料库和高质量出题
第五条线：智能批改和错因诊断
第六条线：多轮卡住辅导 Tutor
第七条线：三端 UI 重构
第八条线：测试体系工程化
```

## 4. 产品形态

### 4.1 孩子端

目标：孩子只看当前一步。

```text
当前任务
→ 开始学习
→ 学习中计时
→ 卡住多轮辅导
→ 完成小测
→ 订正变式题
→ 通过反馈
→ 下一项
```

核心原则：

- 不让孩子看复杂计划。
- 不同时展示太多区域。
- 不让孩子选择跳任务。
- 卡住时给提示和微练习，不直接给答案。
- 通过后给成就反馈。

### 4.2 管理端

目标：家长/管理员三步完成发布。

```text
输入学习安排
→ AI 解析预览
→ 确认发布今日任务
```

高级功能折叠：

```text
AI 配置
资料库
任务源
导入
Agent 日志
```

### 4.3 家长端

目标：第一屏直接回答三件事。

```text
今天学得怎么样？
哪里需要关注？
明天怎么安排？
```

## 5. Agent 架构目标

### 5.1 主控 Agent

```text
LearningManagerAgent
```

职责：

- 接收用户目标和事件。
- 判断当前状态。
- 路由到专业 Agent。
- 执行 guardrail。
- 记录 trace。
- 输出可解释结果。

### 5.2 专业 Agent / Tool

| Agent | 职责 |
|---|---|
| PlanAgent | 解析家长自然语言目标，生成长期计划 |
| ScheduleAgent | 根据计划、画像、复习、时长生成今日任务 |
| MaterialAgent | 检索课本、听写、作业资料 |
| QuizAgent | 根据当天内容生成题目 |
| QuizEvaluatorAgent | 检查题目质量、超纲、泄答案、答案唯一性 |
| GradingAgent | 批改答案，输出错因和能力点 |
| TutorAgent | 卡住时多轮辅导和微练习 |
| ReviewAgent | 生成 D1/D2/D4/D7 补漏复习 |
| MemoryAgent | 写入和召回学生画像 |
| ParentInsightAgent | 生成家长结论和明日建议 |
| GuardrailAgent | 控制安全、范围、时长、权限 |

## 7. 测试目标

测试从脚本型升级为体系型：

```text
unit：规则、评分、状态机
api：接口契约、权限、异常
integration：从发布到完成、多日学习
ui：三端真实浏览器
quality：题目质量、超纲、泄答案
agent_eval：Agent 轨迹、工具选择、记忆召回
security：权限、secret、CORS、记忆注入
```

## 9. 关键约束

- 当前孩子在武汉上小学，内容必须限制在小学五年级上册语文、数学、英语范围内。
- AI 可以建议，但不能无边界自主执行。
- 家长指定的核心计划不能被 Agent 擅自删除。
- 卡住辅导不能直接给作业答案。
- 每日总时长不能无限扩张。
- AI 失败时必须 fallback，并明确标识来源。
- 所有关键输出必须可追溯、可测试、可解释。

## 10. 9.5 版验收标准

```text
1. 家长输入一句话，系统能生成长期计划和今日任务预览。
2. 家长确认后，孩子端能看到唯一当前任务。
3. 孩子学习中计时准确，刷新不丢状态。
4. 孩子卡住后，Agent 进入多轮辅导，并用微练习确认是否学会。
5. 孩子做完后，小测基于当天内容，题目不泄答案、不超纲。
6. 批改后能输出错因、能力点、反馈和变式题。
7. 小测结果能更新学生画像。
8. 次日计划能基于画像动态调整。
9. 家长端能看到今日结论、证据和明日建议。
10. AI 失败时系统仍可用，并显示规则兜底来源。
11. 所有 P0 自动化测试通过。
```
