# 后续使用方式：输入、过程、输出

## 1. 产品定位

后续系统不是让孩子每天和 AI 闲聊，也不是家长每天手动录入任务，而是：

```text
家长设目标和边界
Agent 自动计划、拆任务、陪跑、出题、批改、补漏、总结
孩子每天只按当前任务学习，遇到不会就说，做完就测
家长每天看结论和必要确认
```

推荐形态：

```text
Level 3.5 受控自主学习 Agent
```

## 2. 三类用户职责

| 角色 | 主要职责 | 不应该承担 |
|---|---|---|
| 孩子 | 学习、反馈卡点、完成小测、订正 | 不负责规划、不看复杂设置、不判断长期进度 |
| 家长 | 输入长期目标、设置边界、确认重大调整、看结论 | 不每天手动排任务、不手工出题 |
| Agent | 规划、辅导、出题、批改、诊断、复习、总结 | 不擅自超纲、不直接给作业答案、不越过家长边界 |

## 3. 孩子端使用

### 3.1 孩子的输入

孩子端输入分四类。

#### 点击输入

```text
开始学习
暂停
我卡住了
我会了，继续学
我做完了，开始检查
提交小测
开始下一项
```

#### 简短文本输入

只在卡住时输入一句话：

```text
library 不会读
这道题不知道怎么列式
白鹭这一段不懂
这个词是什么意思
```

#### 答题输入

```text
选择题
填空题
听写题
中译英
数学计算
语文理解
仿写/表达题
```

#### 可选自评输入

```text
简单 / 一般 / 难
我还想再练一道
我需要家长帮忙
```

### 3.2 孩子端过程

```text
打开孩子端
→ Agent 选择当前最该做任务
→ 展示“现在只做这一项”
→ 孩子点开始
→ Agent 计时并记录状态
→ 孩子按步骤学习
→ 卡住则进入多轮 Tutor
→ 学会后回到学习中
→ 做完后进入小测
→ Agent 批改和诊断
→ 通过则进入下一项
→ 未通过则进入订正和变式练习
```

### 3.3 孩子端输出

#### 未开始

```text
现在只做这一项
英语 Unit 1 听读与单词
预计 30 分钟
做到这样算完成：能读出 5 个重点词，并完成小测
[开始学习]
```

#### 学习中

```text
正在学习
已学 08:23
现在要做：先听读课文 5 分钟，圈出不会读的词
[暂停] [我卡住了] [我做完了，开始检查]
```

#### 卡住

```text
你卡在：library 不会读
AI 判断：这是英语单词认读卡点
第一步：把 library 分成 li-bra-ry
试一下：library 的中文意思是什么？
[我会了，继续学] [还是不会，再提示一步]
```

#### 检查

```text
过关小测 1/5
题目：library 的中文意思是？
A. 教室
B. 图书馆
C. 老师
D. 学校
[提交检查]
```

#### 订正

```text
这题错在：词义记忆不牢
先看提示：library 和 book 有关系，是图书馆
再练一道：classroom 的中文意思是？
```

#### 通过

```text
过关！
你完成了：英语 Unit 1 第 1 天
掌握点：school / classroom / library
下一项：语文《白鹭》预习
[开始下一项]
```

## 4. 家长端使用

### 4.1 家长输入

家长不需要每天排任务，只需要输入目标：

```text
五年级上册语文每日一篇课文；
数学小数除法每日一节；
英语 Unit 1 每天听读、单词、默写；
每天总时长不超过 90 分钟。
```

或：

```text
寒假作业本每日一小节；语文书每日一篇课文；做完后必须小测，通过率 80%。
```

### 4.2 Agent 过程

```text
解析自然语言目标
→ 识别学科、教材、章节、节奏、边界
→ 生成长期计划
→ 给家长预览
→ 家长确认发布
→ 每天自动生成今日任务
→ 根据孩子表现动态调整
→ 每晚生成日报
→ 每周生成周报
```

### 4.3 家长输出

家长端第一屏要给结论，而不是数据堆叠：

```text
今日学习结论：整体完成良好，但英语拼写需要补漏

完成：3/4
学习时长：82 分钟
小测通过：2/3
卡住：1 次

重点问题：
1. library 拼写不稳，错了 2 次
2. 数学会算，但步骤表达不完整
3. 语文能复述，但生字听写还要补

明日建议：
1. 先做英语错词默写 10 分钟
2. 数学继续小数除法第 2 节
3. 语文进入下一篇课文，保留 5 分钟生字复习
```

## 5. 管理端使用

管理端应改为三步工作台：

```text
第一步：输入学习安排
第二步：预览 Agent 理解和长期计划
第三步：确认并发布今日任务
```

高级功能默认折叠：

```text
AI 配置
资料库
手动任务源
批量导入
Agent 调试日志
```

## 6. AI 参与度

### 6.1 当前版本 AI 参与度

| 环节 | 当前 AI 参与度 | 当前特点 |
|---|---:|---|
| 家长目标解析 | 50%-70% | AI + 规则混合 |
| 每日任务生成 | 20%-40% | 规则为主 |
| 学习步骤指导 | 20%-40% | 任务源配置为主 |
| 卡住提示 | 60%-80% | AI 有明显价值 |
| 小测生成 | 40%-70% | AI + 题库规则 |
| 小测批改 | 30%-50% | 规则为主，AI 辅助 |
| 错因分析 | 40%-60% | 结构化不足 |
| 复习计划 | 30%-50% | 规则为主 |
| 家长日报 | 50%-70% | AI 可参与总结 |
| 学生画像 | 20%-30% | 记录多于画像 |

当前综合 AI 参与度：`45%-55%`。

### 6.2 目标版本 AI 参与度

| 环节 | 目标 AI 参与度 | 护栏 |
|---|---:|---|
| 目标解析 | 80%-90% | 教材、年级、时长边界 |
| 长期计划 | 75%-85% | 家长确认 |
| 每日排程 | 65%-80% | 不超时、不超纲、复习优先 |
| 学习指导 | 70%-85% | 不直接给答案 |
| 卡住辅导 | 85%-95% | 多轮提示 + 微练习 |
| 小测生成 | 75%-90% | Evaluator 检查 |
| 小测批改 | 60%-85% | 客观题规则，开放题 AI |
| 错因诊断 | 80%-90% | 能力点映射 |
| 复习计划 | 70%-85% | 间隔重复和任务容量 |
| 家长结论 | 85%-95% | 必须有证据 |
| 学生画像 | 70%-85% | 写入门槛和冲突治理 |

目标综合 AI 参与度：`75%-85%`。

## 7. 受控自主学习 Agent 的含义

受控自主不是完全自主。

### AI 可以自主做

```text
分析孩子表现
识别薄弱点
生成计划建议
生成小测和变式题
多轮辅导卡点
批改开放题
更新学生画像
生成家长结论
```

### AI 必须受限制

```text
不能超五年级上册范围
不能直接给作业答案
不能超过每日学习时长
不能擅自删除或跳过家长指定计划
不能低置信度还强行执行
不能让孩子端访问家长/管理端能力
重大调整必须家长确认
```

### 核心公式

```text
AI 负责聪明
规则负责安全
家长负责目标
孩子负责执行
系统负责记录和验证
```

## 8. 最终使用闭环

```text
家长输入目标
→ Agent 生成长期计划
→ 家长预览确认
→ Agent 每天生成今日任务
→ 孩子打开学习驾驶舱
→ Agent 展示当前任务
→ 孩子学习
→ 卡住时 Agent 多轮辅导
→ 做完后 Agent 出小测
→ Agent 批改和诊断
→ Agent 更新学生画像
→ Agent 安排补漏复习
→ 家长端展示今日结论和明日建议
→ 第二天 Agent 根据画像动态调整
```
