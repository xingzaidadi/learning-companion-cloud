# 实施路线图与评分

## 1. 实施原则

不要一口气做成完全自主 Agent。推荐按 6 个阶段推进：

```text
先标准化 Agent 内核
再做学生画像
再做动态排程
再做题目和批改质量
再做多轮 Tutor
最后统一三端 UI 和测试体系
```

## 2. 阶段 1：AgentResult + Tool Registry + Trace

周期：3-5 天。

目标：让所有 Agent 行为有统一输出和日志。

改动：

```text
新增 AgentResult
新增 ToolSpec / ToolRegistry
增强 agent_runs
增加 trace_id
所有 agent.py 方法返回 source/confidence/evidence/warnings
AI 失败记录 fallback_reason
```

验收：

```text
每次计划、出题、批改、卡住、报告都有 trace
页面能显示 AI / 规则来源
AI 异常不影响主流程
```

评分提升：

```text
Agent 7.1 → 7.8
测试 7.4 → 7.8
```

## 3. 阶段 2：学生画像 Memory

周期：4-6 天。

目标：让 Agent 记住孩子的能力变化。

改动：

```text
新增 skill_mastery
新增 memory_records
定义语数英能力点
小测后更新 mastery
卡住后写 episodic memory
家长端展示薄弱点
```

验收：

```text
英语拼写错两次 → 家长端显示英语单词拼写薄弱
数学步骤题错 → mastery 更新到数学.步骤表达
错误事实不写入长期记忆
```

评分提升：

```text
Agent 7.8 → 8.5
家长端 7.2 → 8.3
长期价值 6.8 → 8.4
```

## 4. 阶段 3：动态排程 Planner

周期：4-6 天。

目标：每日任务根据孩子表现动态调整。

改动：

```text
ScheduleAgent 读取 mastery/review/yesterday
每日时长预算
复习优先插队
未完成滚动
低分阻止直接推进新课
发布前预览和家长确认
```

验收：

```text
小测低于 80% → 次日先补漏
昨日未完成 → 今日不重复生成同一新课
今日总时长不超过设置上限
重大调整进入家长确认
```

评分提升：

```text
Agent 8.5 → 8.9
管理端 6.8 → 8.4
产品可用性 8.0 → 8.8
```

## 5. 阶段 4：Quiz Evaluator + GradingAgent

周期：5-8 天。

目标：小测质量和批改可信。

改动：

```text
quiz_items 增加 subject/skill/difficulty/source_ref/quality_score
新增 QuizEvaluatorAgent
新增题目质量评分
开放题 AI + rubric 批改
错因 error_type 结构化
生成变式题
```

验收：

```text
英语题不泄答案
语文听写不暴露答案
数学题答案唯一
开放题有错因和反馈
小测结果更新 mastery
```

评分提升：

```text
Agent 8.9 → 9.2
测试 7.8 → 8.8
孩子端学习价值 8.7 → 9.2
```

## 6. 阶段 5：TutorAgent 多轮辅导

周期：5-7 天。

目标：卡住时真正能解决问题。

改动：

```text
新增 tutor_sessions
新增 tutor_messages
卡点分类
模糊输入追问
一步提示
微练习
解决确认
家长端记录卡点是否解决
```

验收：

```text
孩子说“不会” → Agent 先追问
孩子说“library 不会读” → Agent 给读音拆分和微练习
答对后任务回到学习中
仍不会时可升级家长
```

评分提升：

```text
Agent 9.2 → 9.45
孩子端 9.2 → 9.5
家长端 8.3 → 8.9
```

## 7. 阶段 6：RAG 资料库 + 三端 UI + 测试体系

周期：8-12 天。

目标：整体冲 9.5+。

改动：

```text
material_chunks
资料解析和检索
题目绑定来源
孩子端状态机 UI
管理端三步发布工作台
家长端结论看板
测试目录重构
CI 自动测试
```

验收：

```text
根据五上语数英资料出题
不超武汉小学五上范围
家长端第一屏有结论和明日建议
孩子端任意状态只有一个主动作
P0 测试 100% 通过
```

评分提升：

```text
Agent 9.45 → 9.5+
UI 9.0 → 9.5
测试 8.8 → 9.5
整体 9.5+
```

## 10. 完成后多角度评价

### 测试开发视角

目标分：9.5。

理由：

```text
覆盖主流程、异常、权限、AI fallback、多日学习、记忆污染、状态恢复、题目质量。
```

### UI 设计视角

目标分：9.5。

理由：

```text
孩子端聚焦，管理端三步化，家长端结论优先，三端统一视觉语言。
```

### Agent 开发视角

目标分：9.5。

理由：

```text
有 Manager、Tools、Memory、RAG、Evaluator、Guardrails、Runtime、Trace，属于受控自主学习 Agent。
```

### 产品价值视角

目标分：9.6。

理由：

```text
家长少录入，孩子能独立学，系统能检查和补漏，越用越懂孩子。
```

## 11. 关键风险和控制

| 风险 | 控制 |
|---|---|
| AI 胡编 | RAG + Evaluator + fallback |
| 超纲 | 年级/教材 guardrail |
| 直接给答案 | Tutor 输出护栏 |
| 任务过多 | 每日时长预算 |
| 错误记忆污染 | 记忆写入门槛 |
| 家长不信任 | 结论证据化 |
| 回归频繁 | CI + P0 测试 |
| 框架过重 | 先自研轻量 Agent，不急上 LangChain |

## 12. 最终结论

这个项目适合做完整 Agent 架构，但不适合完全放任自主。最佳路线是：

```text
受控自主学习 Agent
= AI 负责分析和生成
+ 规则负责边界和安全
+ 家长负责目标和确认
+ 孩子负责学习执行
+ 测试负责稳定质量
```

完成本路线后，项目可以从当前约 7.6 分升级到 9.5 分以上。
