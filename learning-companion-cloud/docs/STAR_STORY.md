# 面试 STAR 案例：学习陪跑 Agent 与 Agent Eval

## S：Situation

孩子暑假要提前学习五年级上册语文、数学、英语，目标是新学期考试稳定 95 分以上。传统做法依赖家长手工排计划、盯过程、找题、批改和复盘，工作量大且容易流于形式。

## T：Task

设计并实现一个可本地部署、可接入自有大模型 Key 的学习陪跑 Agent，要求：

- 能根据家长输入自动生成学习计划和今日任务。
- 能基于课本/RAG 资料生成针对性小测。
- 孩子学习过程中支持开始、暂停、卡住、检查、继续下一任务。
- 学完后能沉淀错因、薄弱点和间隔复习计划。
- 项目可作为 AI 测试开发面试作品，必须有 Agent 测评体系和 CI。

## A：Action

- 构建 FastAPI + SQLite + 原生前端的轻量全栈系统，保证可本地运行和可上云。
- 设计受控自主 Agent，而不是完全放任式 Agent，确保孩子学习场景安全可控。
- 增加 RAG 资料导入、知识点图谱、动态策略、D1/D3/D7/D14 复习。
- 增加 `agent_runs` 和 `agent_trace_steps`，记录 Agent 轨迹和工具调用。
- 增加 Function Schema 工具注册表，让 Agent 工具边界可解释、可测试。
- 构建 Eval Harness，覆盖 RAG、计划、卡住、测验、安全、工具调用 89 条 case。
- 接入 GitHub Actions，让 QA Gate 和 Agent Eval 成为持续质量门禁。

## R：Result

- 学习 Agent 能完成从目标输入到每日任务、学习过程、卡点辅导、小测检查、复习闭环的端到端流程。
- Agent Eval 当前 89/89 通过，并输出 JSON/Markdown 报告。
- 项目具备 AI 测试开发面试所需的核心亮点：Agent 架构、RAG、Trace、Eval、CI、安全红队、结果导向闭环。

## 面试亮点表达

> 这个项目不是简单调用 LLM 生成几句话，而是把 Agent 的运行过程拆成可观测、可评估、可回归的工程体系。我把学习目标拆成任务，把任务绑定到时间段，把孩子的卡点和测验结果沉淀成记忆与复习计划，再用 Eval Harness 持续验证 RAG、计划、工具调用和安全行为是否退化。
