# 资深测试工程师全量测试方案

## 1. 测试范围

目标系统：11 岁孩子暑假自主学习陪跑系统，包含管理端、孩子端、家长端、Agent Core、RAG 资料库、小测、卡住辅导、报告和提醒配置。

核心目标：验证系统是否能围绕“五年级上册语文/数学/英语 95+”目标稳定运行，而不是只完成形式化任务。

## 2. 风险模型

| 风险 | 优先级 | 测试重点 |
| --- | --- | --- |
| 孩子端任务状态错乱 | P0 | 开始、暂停、卡住、继续、检查、订正、完成、下一任务 |
| 小测题泄露答案/题目低质 | P0 | 题干不含答案、来源绑定、题型覆盖、评分幂等 |
| RAG 未真正参与 | P0 | 三科真实资料入库、检索命中、source_ref 非空 |
| 管理端发布后孩子端不可见 | P0 | 计划生成、今日任务同步、孩子端 GET |
| 数据/密钥误提交 | P0 | `.env`、`learning.db`、真实 key 不进 Git |
| OCR/PDF 导入失败 | P1 | 文本型 PDF、扫描型 PDF、错误路径、低质量文本 |
| 家长端结论不可用 | P1 | 95+ 看板、薄弱点、日报、周报、调整建议 |
| 权限与角色混乱 | P1 | child/parent/admin 接口访问边界 |
| 异常体验差 | P1 | 无任务、无资料、AI 不可用、重复点击、刷新恢复 |

## 3. 覆盖矩阵

| 模块 | P0 用例 | P1 用例 | 自动化 |
| --- | --- | --- | --- |
| 管理端计划 | 自然语言生成计划、补齐今日任务、手动任务源 | 无效输入、重复生成 | `self_test.py`、`senior_qa_gate.py` |
| 资料/RAG | 语数英覆盖 100%、RAG 检索命中、source_ref 绑定 | URL 导入失败提示、扫描 PDF OCR | `senior_qa_gate.py` |
| 孩子端状态机 | 开始/暂停/卡住/继续/检查/订正/完成 | 刷新恢复、重复点击、下一任务 | `child_flow_integration_test.py` |
| 小测 | 答案不泄露、题型覆盖、评分和订正 | 开放题兜底、错题复习 | `self_test.py` |
| 家长端 | 95+ 看板、日报/周报、薄弱点 | 调整建议、复习建议 | `self_test.py`、`senior_qa_gate.py` |
| UI | 三端打开、按钮可点击、关键文案中文化 | 空态、错误 toast | `ui_click_test.py` |
| 安全 | 密钥/DB 不提交、角色接口不串 | Basic Auth 配置 | `senior_qa_gate.py` |

## 4. P0 验收门槛

1. `/api/materials/coverage` 三科覆盖率必须 >= 0.95。
2. 语文 RAG 必须命中：`日积月累`、`白鹭`、`少年中国说`、`语文园地`。
3. 数学 RAG 必须命中：`小数乘法`、`小数除法`、`多边形的面积`、`验算`。
4. 英语 RAG 必须命中：`Unit`、`Words`、`Listen`、`school`。
5. 今日任务生成后孩子端必须能看到任务。
6. 孩子端任务状态必须满足合法流转，重复点击不能写脏数据。
7. 小测返回给孩子端不能包含 `answer` 或 `explanation`。
8. 数据库私有题目必须有 `source_ref`，不能空。
9. `.env`、`learning.db`、真实 API Key 不得进入 Git。
10. 全量脚本必须通过：`self_test.py`、`validate_agent_core_cases.py`、`ui_click_test.py`、`child_flow_integration_test.py`、`senior_qa_gate.py`。

## 5. 当前结论口径

- 通过全部 P0：可给孩子试用。
- P1 存在缺陷：可试用但需要记录问题。
- 任一 P0 失败：不得给孩子正式使用，必须先修。
